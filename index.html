What Figma is and why you need it basics🔗
Figma is an online editor for creating interface designs. Works directly in the browser, free. Designers make layouts, frontend developers code from them.
Why does a frontend developer need this?
Your manager or designer sends you a Figma link — and you must build exactly the same site from it. Figma shows all sizes, colors, spacing and fonts — directly in pixels. You do not need to guess — just inspect and copy.
1
Designer makes a layout in Figma
Draws buttons, cards, header, colors, fonts — all visual site design.
2
Shares a link with you
You open the file and see exactly the same design. You can view, but not edit.
3
You click an element → see CSS
The right Figma panel shows: size, spacing, color, font, border-radius, and even ready CSS code.
4
Copy styles — paste into code
Click “Copy as CSS” and get ready styles. Only paste them into your own file.
Figma interface — what is where basics🔗
Figma consists of four zones. Remember them — and you will stop getting lost.

Left side. List of all elements: frames, groups, texts, images.
🖼️

Center. The design is drawn here. Zoom: Ctrl+scroll.

💡 Inspect mode (tab in the top right) is made for developers. It immediately shows all CSS values and has a copy-code button.

In Figma, every element is a layer. Layers can have different types:

Export images and icons important🔗
1
Click the needed element (image or icon)
Select the layer with the image or icon in the layers panel or directly on the canvas.
2
Bottom right → the Export section
Click “+” in the Export section. Choose a format: PNG (photos), SVG (icons), WebP (optimization).
3
Click “Export [name]”
The file will download. Put it in your project folder and connect it through <img src="..."> or background-image.
Which format to choose:
PNG — photos and complex images.
SVG — icons and logos. They scale without quality loss.
WebP — like PNG, but 2–3 times lighter. Use it on real websites.
@2x / @3x — for Retina screens. Use 2x for sharp images on Mac/iPhone.

Components, styles and variables PRO🔗
Components are like templates. The designer creates a button once and then reuses it everywhere. For you, this is a signal: make a reusable HTML + CSS block.

Styles are colors, fonts and shadows saved separately. In CSS they become :root { --color-primary: ... }.

Variables are Figma design variables. One variable can change a color everywhere at once. They work like CSS variables in design. When the designer uses variables, you should also use var(--...) in CSS.
💡 See a purple ◆ icon next to a layer? That is a component. Check its name and create a separate CSS class with a similar name.
Workflow for coding from a Figma layout practice🔗
When you get a Figma link, follow this plan and you will not get lost.
1
Study the whole layout
Open Figma. Look at the whole page: what is here? Header, sections, footer. Once you understand the structure, move on.
2
Check how many breakpoints there are
Is there a mobile version? Tablet? There should be several frames nearby. Note their widths.
3
Collect colors and fonts
Open Inspect for any text and note font-family, sizes and colors. Put them into :root as CSS variables.
4
Code top-down, from outside to inside
First the header, then sections, then footer. Inside each section: container first, then its content.
5
Check every element through Inspect
Click in Figma, read CSS on the right, then write it in your file. Do not guess sizes!
6
Pixel perfect? Check with overlay
Open both in nearby tabs. Or use a PixelPerfect plugin: it overlays the layout on top of the page.
Design Tokens: colors, spacing, fonts system🔗
Design tokens are shared design variables. In Figma they are called Variables/Styles; in CSS they are custom properties.
/* CSS variables from the Figma layout */ :root { --color-primary: #7c3aed; --color-bg: #0f172a; --color-text: #e5e7eb; --font-main: Inter, system-ui, sans-serif; --text-sm: 14px; --text-base: 16px; --text-xl: 24px; --space-2: 8px; --space-3: 12px; --space-4: 16px; --radius-sm: 8px; --radius-md: 12px; } .button { background: var(--color-primary); padding: var(--space-3) var(--space-4); border-radius: var(--radius-sm); }
Why tokens: if the main color changes in the layout, you change one variable instead of 40 classes.
What to extract: main colors, text sizes, spacing, border radii, shadows, container width.
How to work: first collect tokens from Figma, then code components through var(--...).

🟢 Node.js — JavaScript on the server

JavaScript used to live only in the browser. Node.js lets you run JS on the server, so now one language can be used for both frontend and backend. This is your next step after becoming confident with JavaScript. Node.js is the foundation for servers, APIs, chats, and websites that handle many users.

⚠️ Do not start Node.js until you are confident with regular JS. Node uses the same JavaScript, but on the server. First learn functions, loops, arrays, and objects in the browser. Then Node will feel much easier.

🗄️ SQL — databases

localStorage keeps data only in one browser. To store data reliably, you need a database. SQL is the language used to query databases. Learn it and you can store users, posts, orders, and much more. It is an essential backend skill.

🖥️ Servers, cache, balancing, Docker

This is what lets a website handle many users. It is less about ordinary programming and more about systems engineering: how servers work, how to improve performance, and how to scale. It is an advanced topic, but useful for understanding where to grow.

⚠️ This is a distant goal, not for now. First frontend, then Node.js, then databases, and only after that this topic. This section is an overview so you can understand the big picture and where the path leads.
PostgreSQL — powerful relational DB
PostgreSQL is a standard for serious projects. Open source, ACID-compatible, JSON support, full-text search. Used everywhere from startups to banks.
SQL ACID JSON Indexes

Direction of the main axis. The default is row (horizontal).

Alignment along the main axis (horizontal in a row).
align-items
stretch | center | flex-start | flex-end | baseline
Alignment along the cross axis (vertical in a row).

nowrap | wrap | wrap-reverse
Controls whether child elements wrap onto a new line.
gap
gap: 16px | gap: 8px 16px
Spacing between elements. Replaces margins on child elements.
align-content
flex-start | center | space-between | stretch
Alignment of lines when flex-wrap is set to wrap.
flex-flow
flex-flow: row wrap
Shorthand for flex-direction and flex-wrap.
On the element (child)
flex
flex: 1 | flex: 0 0 200px
Grow, shrink, and basis. flex: 1 takes all available space.
flex-grow
flex-grow: 1
How much an element grows relative to others.
flex-shrink

0 means do not shrink. 1 means shrink (default).

Base size before grow and shrink are applied.
align-self
auto | center | flex-start | stretch
Overrides align-items for a specific element.
order
order: -1 | 0 | 1
Display order. The default is 0; smaller values appear first.
On the container (parent)
